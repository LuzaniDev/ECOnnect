# ECOnnect v1.2.0

**Data**: 2026-06-12

## Novidades

### Sistema de Temas (Theme Manager)
- Novo sistema centralizado de temas em `frontend/app/core/theme.py`
- Suporte a **múltiplos temas**: Black (escuro), White (claro) e Matrix (verde)
- Troca dinâmica de temas com `ThemeManager` e sinal `theme_changed`
- Aplicação via `QPalette` para consistência entre widgets nativos
- Sidebar QSS gerado dinamicamente conforme o tema ativo
- Persistência da escolha em `~/.econnect/theme.json`

### Módulo Meta (WhatsApp Business API)
- Novas tabelas no banco: `meta_credentials` e `meta_messages`
- Modelos `MetaCredentials` e `MetaMessage` com SQLAlchemy
- Rota `/api/meta/*` para gerenciamento de credenciais e envio
- Webhook `/api/webhook/*` para receber atualizações de status do Meta
- Templates de requisição com campos `meta_template_id` e `meta_status`
- View `MetaView` e `MetaCredentialsView` no frontend
- API client `frontend/app/api/meta_api.py`

### Gerenciamento de Permissões por Aba
- Coluna `tab_permissions` (JSON) na tabela `users`
- Endpoint `PUT /api/users/{id}/permissions` para admin
- Sidebar filtra abas visíveis conforme `tab_permissions` do usuário
- `AdminTabsView` para gerenciamento visual das permissões
- Agrupamento da sidebar por categorias (Navegação, Mundo dos Bots, Administração)

### Dashboard Expandido
- Requisições por dia com detalhamento por status (sent/pending/cancelled) — últimos 14 dias
- Top 10 templates por volume de requisições
- Requisições por tag
- Top 5 usuários por requisições
- Comparação semanal (esta semana vs. semana passada) com breakdown por status
- Distribuição por tipo de integração
- Auditoria recente aumentada de 5 para 10 registros

### Módulo Cobrança (Backend)
- Rota `POST /api/cobranca/check-sent` para verificação de envio por telefone
- `CobrancaService` com verificação de cooldown por usuário
- Consulta de último envio com tag "cobrança" nos últimos N dias

### Requisições como View Independente
- `RequisicoesView` movida para view própria no frontend
- Sidebar com navegação direta para a aba
- Integração com lista e formulário de templates como views separadas

### Gerenciamento de Variáveis SQL
- `VariablePickerDialog`: listagem e busca de variáveis disponíveis
- `SqlVariableManagerDialog`: CRUD completo de variáveis SQL
- `SqlVariableEditDialog`: editor com teste de SQL em tempo real no Firebird
- Parse automático de colunas SELECT e seleção de coluna de valor
- Suporte a parâmetros `dict` no Firebird client

### Diálogo de Agendamento
- `ScheduleDialog` com opções "Enviar agora" e "Agendar para"
- Seletor de data/hora com calendário popup
- Exibição de quantidade de clientes e template selecionado

## Melhorias
- **Dashboard**: análise expandida com gráficos por template, tag, usuário; comparação semanal; distribuição por tipo de integração
- **Firebird client**: suporte a parâmetros `dict`; `conn.commit()` após queries para evitar locks; rollback adequado em erro; logging de queries
- **Config**: suporte a variável `ECONNECT_PORT` para configuração da porta da API
- **Worker**: keepalive para objetos relay evitando coleta prematura; cleanup automático após 5s; try/except no `relay.error.emit`
- **Sidebar**: dinâmica com agrupamento por categoria e filtro por permissões
- **Logout**: retorna à tela de login em vez de fechar o aplicativo
- **Tema**: centralizado em `core/theme.py` com `ThemeManager` — `styles/theme.py` mantido apenas como compatibilidade
- **Segurança**: valores padrão de Firebird (`SYSDBA`/`masterkey`) removidos — exigem configuração explícita
- **Sidebar QSS**: gerado dinamicamente pelo tema em vez de fixo

## Correções
- **Firebird locks**: adicionado `conn.commit()` após `fetchall()` em queries SELECT
- **Worker crash**: erro no `relay.error.emit` tratado com try/except para evitar falha na thread
- **Frameless window**: removido — causava problemas de redimensionamento e compatibilidade; retorno à janela nativa do Windows
- **Valores padrão Firebird**: `fb_database`, `fb_user` e `fb_password` agora vazios por segurança

## Breaking Changes
- **Janela frameless removida**: ECOnnect volta a usar janela nativa do Windows com moldura padrão
- **Valores padrão Firebird removidos**: configurar Firebird manualmente nas Configurações
- **Tema refatorado**: `styles/theme.py` não é mais a fonte primária de tema; usar `core/theme.py`

## Procedimentos de Atualização
1. Migrações automáticas executadas na inicialização:
   - Coluna `tab_permissions` (JSON) adicionada à tabela `users`
   - Colunas `meta_template_id` (VARCHAR) e `meta_status` (VARCHAR) adicionadas à tabela `templates`
   - Tabela `meta_credentials` criada (waba_id, phone_number_id, access_token)
   - Tabela `meta_messages` criada (from_phone, to_phone, template_name, status)
2. Arquivo `~/.econnect/theme.json` criado automaticamente na primeira execução
3. Reconfigurar credenciais Firebird nas Configurações (valores anteriores foram resetados)
4. Nenhuma ação manual adicional necessária

## Dependências
- Python 3.10+
- PySide6 (Qt for Python)
- FastAPI + Uvicorn
- Firebird 2.5+ (opcional, para integrações)
- PostgreSQL 12+
- Requests (httpx para API client)
