# v1.5.1 - Refactor: Scan Inicial, Juros, Preview SVGs, PDF UNC Fix

**Data**: 2026-07-09

## Alteracoes

### Scan inicial de boletos (app.py)
- Removida verificacao previa `verificar_scan_necessario()` que consultava os 10 ultimos PDFs
- Agora pergunta diretamente: "Deseja processar os ultimos X boletos agora?"
- Mensagem mais objetiva, sem emojis
- Processamento so ocorre se usuario clicar em Sim

### Correcao do calculo de juros e multa (mundo_bots.py)
- Adicionado `LEFT JOIN TRecParametro Pmt` para resgatar `TipoJuro` e `Multa` do banco
- `TipoJuro` estava sempre vazio (`''`), fazendo o calculo Python sempre resultar em 0
- `Multa` estava sempre 0, nunca era aplicada
- `Juros` agora tem fallback para `Pmt.Juros` quando `Cli.JurosAtraso` for 0
- O calculo continua 100% em Python (não foi movido para SQL)

### Correcao da previa do envio (mundo_bots.py)
- **Juros**: indice da coluna corrigido de `row[24]` (IDTRECPARCELA) para `row[30]` (ValorJuros)
- **Multa**: adicionada exibicao do valor da multa (`row[31]`)
- Na pratica, antes mostrava o ID da parcela como se fosse juros (ex: "R$ 50.000,00")

### Emojis substituidos por SVGs na previa
- 📱 -> Phone SVG (Lucide)
- 👤 -> User SVG
- 💰 -> Dollar-sign SVG
- 📅 -> Calendar SVG
- ⏱ -> Clock SVG
- 📊 -> Trending-up SVG (juros)
- 📋 -> Clipboard SVG (ja existia no codigo)
- 🚀 -> Send SVG (fluxo)

### Abertura de PDF em rede (UNC)
- Substituido `os.startfile()` por `subprocess.Popen(["explorer", pdf_caminho])`
- `os.startfile()` falha com caminhos UNC (`\\SRVAPP\...`) em diversas configuracoes
- Explorer.exe tem o contexto de rede necessario para acessar shares
