## Correcoes

### Path normalization no Configurador
- Adicionado metodo `_norm_path()` que converte `\` para `/` ao salvar `FB_DATABASE`
- Evita que python-dotenv interprete `\d`, `\e`, `\T` como escapes (ex: `C:\ecosis\dados` virava `C:ecosisdados`)
- Tooltip no campo FB_DATABASE explicando formato correto para servidor remoto (`servidor:C:\path`)

### Forward-slash no firebird_client.py
- `conectar()` agora converte `/` para `\` no DSN antes de chamar `fdb.connect()`
- Compatibilidade com paths salvos em forward-slash no `.env`
- Garante que `srvlubri:C:/ecosis/dados/ecodados.eco` funcione corretamente

### .env.example atualizado
- `FB_DATABASE` agora tem valor de exemplo com forward slashes (`C:/ecosis/dados/ecodados.eco`)
