# v1.4.0 - ECOnnect Configurador + PyInstaller Onefile + Correcao de Imports

**Data**: 2026-07-07

## Novidades

### ECOnnect Configurador (novo executavel separado)
- Novo `ECOnnectConfigurador.exe` (52 MB), construido com PyInstaller onefile
- Janela PySide6 para configurar `.env` antes de rodar o ECOnnect principal
- Teste de conexao PostgreSQL (via asyncpg)
- Teste de conexao Firebird (via fdb)
- Criacao automatica de banco/usuario PostgreSQL (superuser fallback)
- Migracao opcional de tabelas PostgreSQL (Base.metadata.create_all)
- Migracao opcional da tabela BOLETO_GERADO no Firebird
- Secao "Boleto PDF" opcional com instrucoes
- Geração de JWT_SECRET automatica
- Botao "Salvar e Abrir ECOnnect" que ja dispara o app principal
- Log em tempo real das operacoes dentro do configurador
- `.spec` minimalista: exclui numpy, PIL, lxml, cryptography, QtWebEngine e outros
- Dependencias inclusas: PySide6 (widgets only), asyncpg, fdb, sqlalchemy

### PyInstaller Onefile (ECOnnect.exe)
- Build alterado de one-dir para one-file: unico `ECOnnect.exe` (266 MB)
- Sem pasta de dependencias ao lado do executavel
- `.env` sempre lido do diretorio do executavel (nunca do bundle MEIPASS)
- `frontend/app/config.py` - `_resolve_env()` sempre retorna `exe_dir/.env` quando frozen
- `backend/app/config.py` - `_resolve_env_file()` sempre retorna `exe_dir/.env` quando frozen, sem fallback

## Correcoes

### Circular import PySide6 + pydantic (causa raiz do servidor nao subir)
- Adicionado `os.environ["PYI_SIGNATURE_MODULE"] = ""` antes de qualquer import
- Resolve `ImportError: cannot import name 'import_string' from partially initialized module 'pydantic._internal._validators'`
- Bug conhecido do shibokensupport (PySide6) com pydantic/fastapi em frozen environment

### stderr None no frozen
- Adicionado `if sys.stderr is None: sys.stderr = io.StringIO()`
- Resolve `ValueError: Unable to configure formatter 'default'` no uvicorn
- Uvicorn tentava acessar `sys.stderr.isatty()` que quebrava com stderr None

### Missing passlib.handlers.pbkdf2
- Adicionado `passlib.handlers.pbkdf2` aos hiddenimports do spec
- `auth_service.py` usa `CryptContext(schemes=["pbkdf2_sha256"])` que depende deste handler

### numpy.typing crash no PyInstaller
- Adicionado `'numpy.typing'` aos excludes do spec
- OpenBLAS crashava o subprocesso do PyInstaller durante analise de dependencias

### Log extremamente verboso
- Marcadores `[ENV]`, `[CONFIG]`, `[DB]`, `[LIFESPAN]`, `[SERVER]`, `[MIGRATION]` em todo o startup
- Timestamp e duracao de cada etapa do servidor
- Traceback COMPLETO nas excecoes (nao apenas `str(e)`)
- Validacao de .env campo a campo com log de cada valor (senha mascarada)

### Startup mais rapido
- `DB_RETRIES` reduzido de 5 para 1 (ja configurado e testado)
- `wait_for_server()` com poll a cada 0.2s e deteccao de falha via `threading.Event`
- Se servidor falha, aborta em milissegundos (nao espera 120s)

### data_pipeline removido (era experimental)
- `frontend/app/views/data_pipeline.py` removido
- Sidebar sem referencia ao Data Pipeline
